# API コントラクト

dashboard のバックエンド API エンドポイント定義。

## 設計方針

- REST API ベース
- JSON 形式でのやり取り
- ローカル起動を前提とするため、初期フェーズでは認証なし（将来拡張）

## エンドポイント一覧

### 投稿関連

- 投稿一覧取得（アカウント・期間・型でフィルタ可能）
- 投稿詳細取得
- 投稿の最新メトリクス取得

### メトリクス関連

- メトリクス時系列取得（特定投稿）
- メトリクス手動入力
- 型別集計取得

### アカウント関連

- アカウント一覧取得
- アカウントごとのサマリー取得(期間別フォロワー数、投稿数等）

### ドラフト関連（オプション）

- ドラフト一覧取得
- ドラフト詳細取得

## レスポンス形式

各エンドポイントは以下の形式でレスポンスを返す:
- 成功時: データ本体 + メタデータ
- エラー時: エラーコード + エラーメッセージ
