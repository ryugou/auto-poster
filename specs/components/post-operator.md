# post-operator

**構築フェーズ**: Phase 4

drafts DB のレビューと、投稿情報の記録を行うCLIツール。

## 責務

- drafts の一覧・詳細表示
- レビュー結果（承認 / 却下）の記録
- 投稿後の投稿URL・投稿時刻の記録
- drafts・posts のライフサイクル管理

## 非責務

- X への投稿アクション（人間が手動投稿）
- ドラフト生成（post-generator の責務）
- メトリクス入力・可視化（dashboard の責務）

## 入力

- drafts DB のレコード
- 人間オペレータのCLI操作

## 出力

- drafts DB のレビューステータス更新
- posts DB への投稿記録

## 主な機能

- ドラフト一覧・詳細表示
- レビュー操作（承認 / 却下 / 編集）
- 投稿記録登録（投稿URL・投稿時刻）
- 投稿予定時刻の表示
- 投稿予定時刻の手動上書き（レビュー時に必要なら変更可能）

## オペレーションフロー

```
1. post-operator 起動
2. 未レビュードラフト一覧を確認
3. レビュー判断（承認 / 却下）
4. 承認したドラフトを人間がXに手動投稿
5. 投稿URL・投稿時刻を入力
```

## データライフサイクル管理

- drafts のステータス更新（承認・却下）
- posts の作成
- 却下ドラフトの自動削除方針は実装判断

## エラーハンドリング

- DB更新失敗: エラー表示、再試行可能
- 不正な投稿URL入力: バリデーションで検知
