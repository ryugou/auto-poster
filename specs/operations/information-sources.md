# 情報源定義

各アカウントで使用する外部情報源を定義する。

## A: AI新着まとめ

### 主情報源

#### xAI Grok API
- 目的: X上で話題のAIツール・サービスを日次で取得
- 取得内容: 話題のAI関連情報（公式発表、バズ投稿、新ツール発見）
- クエリ方針: 「今日話題のAIツール」「過去24時間で発表された新AI関連情報」等
- 複数クエリで網羅性を確保し、重複排除する

### 補助情報源

初期は Grok API 単独で運用。情報密度が不足した場合に以下を追加検討:
- Product Hunt の日次新着
- 主要AI企業の公式発表
- Hacker News のAI系上位エントリ

## B: マンガ新刊速報

### 主情報源

#### 楽天ブックスAPI
- 目的: 新刊情報、価格、ランキング、予約情報の取得
- 利用前提: 楽天アフィリエイト登録
- 制約: API利用制限の範囲内で日次取得

#### コミックナタリー RSS
- 目的: アニメ化・実写化発表、話題性情報の補完
- 利用前提: RSS購読のみ（認証不要）

### 補助情報源

#### Amazon PA-API（Phase 6 以降）
- 目的: Amazon Kindle のセール情報、ランキング
- 利用前提: Amazonアソシエイト登録 + 適格販売実績
- 導入タイミング: アソシエイト利用条件を満たしてから

## 情報源共通の扱い

### 取得頻度
- 日1回を初期想定
- 情報源の更新頻度に応じて個別調整

### 正規化
- 情報源ごとに異なるレスポンス形式を共通スキーマに正規化
- 共通スキーマの詳細は [../components/dashboard/data-model.md](../components/dashboard/data-model.md) を参照

### 重複検出
- 同一素材が複数情報源から取得される可能性あり
- 重複判定して raw_materials への重複保存を避ける

### エラー時
- リトライはせず、次回定期実行に任せる
- エラー内容はログに記録

## 情報源選定の原則

### 採用する情報源
- 公開情報を扱っていること
- API または RSS で構造化取得できること
- 利用規約を遵守できること

### 採用しない情報源
- スクレイピングが規約違反になる可能性があるサイト
- 有料で高額な情報ソース（コスト割高）
- 情報の信頼性が低いサイト
