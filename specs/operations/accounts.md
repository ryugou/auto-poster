# アカウント定義

本システムが運用するXアカウントの定義。

## アカウント一覧

| ID | アカウント名 | ハンドル | 領域 |
|---|---|---|---|
| A | AI新着まとめ | @ai_shinchaku | 話題のAIツール・サービスの新着情報 |
| B | マンガ新刊速報 | @manga_shinkan | 新刊コミック・セール・話題作情報 |

## 共通設定

### ターゲット層
受動的に情報を受け取りたい一般ユーザー。詳細は [../CLAUDE.md](../CLAUDE.md) の「ターゲット層」を参照。

### 投稿頻度
各アカウント日1〜3投稿。

### 投稿型
5型を使い分ける（速報 / 驚き / 実用解説 / まとめ / インパクト衝撃）。詳細は [post-templates.md](./post-templates.md)。

### 情報源
アカウントごとに異なる。詳細は [information-sources.md](./information-sources.md)。

## アカウント A: AI新着まとめ

### プロフィール文(初期案）
```
毎日話題のAIツール・サービスを厳選してまとめます📌
ChatGPTの次に来るAI、見逃したくない人向け。
新着情報 / 使い方 / 無料で使えるAI
#AI #ChatGPT #生成AI
```

### 5ステップパイプラインの領域固有定義

#### Step2: 分解
取得したAI関連情報を以下の要素に分解する:
- **ツール・サービス名**
- **提供元**
- **できること（機能）**
- **料金（無料 / 有料 / フリーミアム）**
- **公開日・発表日**
- **話題性の指標**（X上の言及数、バズ度など）

#### Step3: 素材選別
以下を満たす素材を投稿候補とする:
- 話題性が一定以上（基準値は運用中に調整）
- 一般ユーザーが使える、または使い方を想像できる
- 事実として確認可能

以下は除外:
- 研究論文レベルの抽象的情報
- エンジニア以外には価値がわからない技術的深掘り
- 事実確認が取れない情報

### 情報鮮度・生成頻度

- 情報鮮度要件: 数時間以内
- info-collector 実行頻度: 日3回（投稿時間帯の直前）
- post-generator 実行頻度: info-collector 直後

### 投稿時間帯

- 朝枠: 7-9時
- 昼枠: 12-13時
- 夜枠: 21-23時

## アカウント B: マンガ新刊速報

### プロフィール文(初期案）
```
新刊発売日・無料キャンペーン・話題作を毎日配信📚
紙・電子どちらも対応。
推し作品の新刊を絶対逃さない人向け。
#漫画 #マンガ #新刊 #Kindle #電子書籍
```

### 5ステップパイプラインの領域固有定義

#### Step2: 分解
取得した漫画関連情報を以下の要素に分解する:
- **作品タイトル**
- **巻数**
- **作者**
- **出版社・レーベル**
- **発売日**
- **価格**
- **電子書籍の有無**
- **セール情報**（対象か、割引率、期間）
- **話題性の指標**（アニメ化・実写化の有無、ランキング順位など）

#### Step3: 素材選別
以下を満たす素材を投稿候補とする:
- 話題性が一定以上（ランキング上位、アニメ化発表、セール対象など）
- 一般読者に知名度がある、または知名度が上がりそう
- 事実として確認可能

以下は除外:
- 同人誌・成人向け（アカウント規約違反リスク）
- 特定読者層に限定されすぎる作品
- 発売日が未定・不確定な情報

### 情報鮮度・生成頻度

- 情報鮮度要件: 発売日当日内
- info-collector 実行頻度: 日1回(朝)
- post-generator 実行頻度: 日3回（投稿時間帯の直前）

### 投稿時間帯

- 朝枠: 7-9時
- 昼枠: 12-13時
- 夜枠: 21-23時

## アカウント固有の注意点

### A: AI新着まとめ
- 有料プロモーションを示唆しない（広告案件の場合は明示）
- 特定ツールを過度にディスらない
- AI生成であることを示唆する表現は避ける

### B: マンガ新刊速報
- ネタバレ禁止（表紙・あらすじ公式範囲まで）
- 作者・作品への誹謗中傷禁止
- 海賊版サイトへの言及・リンク禁止
