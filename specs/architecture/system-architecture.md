# システムアーキテクチャ

## 概要

本システムは Rust 製のバックエンドに全機能を集約し、シングルバイナリとして配布する。内部的には複数のコンポーネント（info-collector / post-generator / post-operator / dashboard）が SQLite データベースを介して疎結合に連携する。ダッシュボード UI は同一バイナリに埋め込む。

## コンポーネントの言語選定

言語選定の詳細は [tech-stack.md](./tech-stack.md) 参照。全コンポーネントは Rust で実装し、単一バイナリにまとめる。ダッシュボード UI（React 等）のビルド成果物はこのバイナリに埋め込む。

本プロジェクトではグローバル CLAUDE.md の「常駐はRust、スクリプトはTypeScript」ルールを明示的に上書きする。理由は、ダッシュボードが無くてもバックエンド単体で運用が完結することを要件とするため、言語を Rust に統一する必要があるため。

## データベース

- SQLite単一ファイル
- 全コンポーネントが同一DBに接続
- スキーマ詳細: [components/dashboard/data-model.md](../components/dashboard/data-model.md)

## シークレット管理

- 1Password CLI + direnv でシークレットを環境変数として注入
- コミット対象ファイルにシークレット値を含めない

## Docker構成

全コンポーネントをDocker上で動作させる。
