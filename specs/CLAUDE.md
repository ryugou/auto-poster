# X自動運用システム

2つのX（旧Twitter）アカウントを並行運用し、定点観測型の情報発信でフォロワー獲得を目指すシステム。

## アカウント
- **AI新着まとめ** (@ai_shinchaku): 話題のAIツール・サービスの新着情報
- **マンガ新刊速報** (@manga_shinkan): 新刊コミック・セール・話題作情報

## ターゲット層

受動的に情報を受け取りたい一般ユーザー。以下の行動特性を持つ:
- 自分で情報を追う時間・気力がない
- 「フォローしておけば情報が流れてくる」を求める
- 深い分析より見出しレベルの情報で満足する
- 画像・スクショ・断定的評価に反応する
- エンジニア層のような能動的情報収集層は対象外

## システム責務

本システムは以下の機能を提供する。実装ロードマップは [roadmap.md](./roadmap.md) 参照。

1. 情報収集: 外部ソース（Grok API、楽天ブックスAPI、RSS）から日次で素材取得
2. 投稿生成: 5つの型テンプレに沿って投稿ドラフトを生成
3. 投稿管理: ドラフト保管、レビュー、投稿記録
4. 分析: 投稿の反応データを記録、型別の勝ち負けを可視化

## 前提

### 開発環境
- Docker (OrbStack) のみ。ローカルnpm/cargo install禁止
- 言語: Rust（バックエンド全機能。情報収集・投稿生成・投稿オペレーション・メトリクス取得までを単一プロセス体系に収める）
- 配布: シングルバイナリ。ダッシュボード UI もこのバイナリに embed
- ダッシュボード UI: React 等の任意フレームワーク（実装時に選定）
- DB: SQLite
- シークレット管理: 1Password CLI + direnv
- Conventional Commits（日本語）

### 外部サービス
- X Developer Portal: 既存登録状況は未確認。新規登録の場合、2026年2月以降は従量課金制で最低$5のチャージが必要
- Grok API: xAI APIアカウント必要
- 楽天ブックスAPI: 楽天アフィリエイト登録必要
- 投稿アクション: Phase 4-5 は人間が手動投稿、Phase 6 で X API 連携に切り替え予定

## NGルール

### 投稿内容
- 他社・他人を貶める投稿禁止
- 医療・金融の具体的推奨（「この薬が効く」「この株を買え」等）禁止
- 政治的・宗教的な立場表明禁止

### 著作権
- 漫画の表紙画像: 出版社が公式に配布している商品画像（Amazon商品画像等）は利用可。ファン撮影画像やスキャン画像は利用不可
- AIツールのスクリーンショット: 公式サイト掲載のスクリーンショットは利用可。ユーザー環境のスクショを転載する場合は著作権者の明示的許可が必要
- 他アカウントの投稿転載: 禁止。引用RTは可

### システム運用
- SIVIRA本業情報・個人情報の混入禁止
- 未確認情報の速報禁止（誤報リスク）
